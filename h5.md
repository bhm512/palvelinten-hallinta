# h5 

Tiivistelmä

- git add; lisää kaikki hakemistossa tehdyt muutokset Gitin staging-alueelle, jotta ne voidaan sisällyttää seuraavaan commit-versioon. (https://git-scm.com/docs/git-add)
- git commit; Tallentaa staging-alueen muutokset paikalliseen Git-historiaan uutena commitina. Commit on projektin versiotilannekuva. (https://git-scm.com/docs/git-commit)
- git pull; Hakee etävarastosta uusimmat muutokset ja yhdistää ne paikalliseen työhösi. (https://git-scm.com/docs/git-pull)
- git push: Lähettää paikalliset commitit etäpalvelimelle. Näin muutokset näkyvät GitHubissa ja muille käyttäjille. (https://git-scm.com/docs/git-pull)
- Komento kloonaa varaston, ja sen jälkeen näyttää koko Git-historian sekä commit-kohtaiset muutokset (diffit) värillisenä, helposti selattavassa muodossa komentorivillä. (https://github.com/terokarvinen/suolax/)
- Git on versiohallintajärjestelmä, joka eroaa monista muista vanhemmista työkaluista. Vaikka Gitin käyttö saattaa vaikuttaa tutulta, sen sisäinen toimintatapa on täysin erilainen.
- Git ei ajattele tietoja tiedostojen muutoksina ajan yli, vaan se tallentaa kokonaisia projektin tilannekuvia (snapshoteja) jokaisesta commitista.
